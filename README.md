# psychology-quiz

`psychology-quiz` — Telegram-бот викторины по психологии для учебного использования. Текущий продуктовый контур: только **Module 1**. Бот работает в режиме **long polling** (без webhook), **Web UI отсутствует**, внешняя генерация вопросов во время работы (RAG/retrieval) отсутствует.

## Текущий продуктовый контур (Module 1)

Активные категории:
- Введение в профессию
- Общая психология
- Физиология человека
- Физиология ВНД
- Психофизиология

## Режимы викторины (UX v2)

Поддерживаются три режима запуска:
- `Конкретная тема`
- `Микс из выбранных тем`
- `Все темы`

Краткий сценарий:
- `Конкретная тема` → выбор одной категории → выбор количества вопросов → выбор сложности.
- `Микс из выбранных тем` → мультивыбор категорий → `Готово` / `Сбросить` → выбор количества вопросов → выбор сложности.
- `Все темы` → выбор количества вопросов → выбор сложности.

Для режима `Микс из выбранных тем` выбранный набор категорий сохраняется на уровне сессии. Сценарий `Пройти еще раз` повторяет тот же набор выбранных тем, а не пересобирает его случайно.

## Вспомогательный UX

- постоянная клавиатура reply keyboard в личном чате (`🎯 Начать викторину`, `ℹ️ Помощь`, `👁 Режим чтения`, `🙈 Скрыть меню`)
- ручное скрытие меню через `🙈 Скрыть меню`
- восстановление меню через inline-кнопку `👁 Показать меню`
- резервное восстановление через `/start`
- режим чтения: `Обычный` / `Бионическое чтение`

## Поток данных: JSON → seed → SQLite

- Основной источник данных банка вопросов: JSON в `content/questions/module1/`.
- SQLite **не** является основным источником истины: это слой хранения и выдачи данных во время работы бота.
- Заполнение и обновление БД выполняется сидером `scripts/seed_questions.py`.
- Операционная цепочка обновления контента:
  1. изменить JSON;
  2. сделать PR;
  3. после merge запустить seed;
  4. рабочая SQLite получает обновленные данные.

## Коротко про хранение данных

- Путь к SQLite задается через переменную `DB_PATH`.
- Путь по умолчанию: `/data/quiz.sqlite3`.
- При изменении JSON-контента нужно запускать seed через `scripts/seed_questions.py`.
