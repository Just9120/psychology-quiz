# ТЗ: psychology-quiz (рабочий product/content spec)

## 1. Назначение проекта
- Проект — Telegram-бот викторины по психологии.
- Формат — учебный продукт с вопросами по темам психологии.
- Текущий приоритет разработки контента: module 1.

## 2. Текущий продуктовый scope
- В проде поддерживается только module 1.
- Категории module 1:
  - Введение в профессию
  - Общая психология
  - Физиология человека
  - Физиология ВНД
  - Психофизиология
- RAG в текущей версии отсутствует.
- Runtime-генерация вопросов отсутствует.
- Web UI отсутствует.
- Режим работы бота: long polling.
- Хранение: SQLite (runtime) + JSON-файлы (контентный источник).

## 3. Архитектурный принцип данных
- Source of truth для question bank: JSON-файлы в репозитории.
- SQLite не является источником истины; это runtime storage / serving layer.
- Пополнение БД выполняется штатным сидером: `scripts/seed_questions.py`.
- Корректная цепочка изменения контента:
  1. изменить JSON в репозитории;
  2. сделать PR;
  3. после merge запустить seed;
  4. SQLite получает актуальные вопросы.

## 4. Структура question bank
- Основная директория module 1: `content/questions/module1/`.
- Каждая крупная категория хранится в отдельном JSON-файле.
- Формат элемента массива вопроса:
  - `id`
  - `category`
  - `source_ref`
  - `difficulty`
  - `status`
  - `question`
  - `options`
  - `correct_option_index`
  - `explanation`
- Правила `id`:
  - уникальность в рамках банка;
  - консистентный префикс категории и порядковый номер.
- Правила `category`:
  - единое написание внутри файла;
  - именование должно совпадать с продуктовой категорией.
- В БД загружаются только вопросы со статусом `approved`.

## 5. Контентные правила
- Вопросы формируются только по project sources, доступным в проектном контурe.
- Интернет не используется как первичный источник контента для question bank.
- Если вопрос не получается привязать к source grounding, вопрос не включается.
- Внутри категории соблюдать баланс:
  - базовые понятия;
  - механизмы;
  - прикладные мини-кейсы.
- Практический слой не выносится в отдельную bot-category: прикладные вопросы встраиваются в профильные темы.

## 6. Текущий content scope module 1
- Введение в профессию
- Общая психология
- Физиология человека
- Физиология ВНД
- Психофизиология

## 7. Принципы развития банка
- Сначала усиливается покрытие module 1, затем расширяется следующий модуль.
- Новые категории добавляются отдельными JSON-файлами в `content/questions/module1/`.
- Приоритет: качество формулировок и source grounding, затем увеличение количества.
- Без необходимости не смешивать крупные UX-изменения и крупные контентные PR в одном наборе изменений.

## 8. Operational workflow
- Code-only deploy нужен, когда меняется только логика приложения без изменения контента.
- Seed обязателен, когда меняется question bank (добавление/правка JSON с вопросами).
- SQLite не редактируется вручную, чтобы не расходиться с версионируемым source of truth в Git.

## 9. Ограничения и анти-паттерны
- Нельзя использовать ручное редактирование SQLite как процесс обновления контента.
- Нельзя добавлять хаотичные вопросы без внятных `source_ref`.
- Нельзя смешивать PM-layer/нерелевантные домены в psychology question bank.
- README не используется как полное ТЗ; для рабочих правил применяется этот документ.

## 10. Ближайший roadmap
1. Довести покрытие категории «Психофизиология» до целевого качества и консистентности.
2. Далее расширять банк категории «Общая психология».
3. Затем наращивать банк категории «Физиология ВНД».
